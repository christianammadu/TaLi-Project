### WP-02 — Prompt arrays + compound example + token bump

- **Goal:** Unified JSON schema emits arrays; add the "bought 6 … and sold 4 …" worked example; raise `max_tokens` 300→~700.
- **Files:** `app/services/nlp.py`
- **Depends on:** WP-01 (shape).
- **DoD:** live message *"Bought 6 bags of rice at 400 per one and sold 4 for 6000"* returns JSON with 2 transactions + 2 inventory entries; no truncation.

### WP-03 — Ledger loop with per-item `event_id`

- **Goal:** Convert the three single `if` blocks to loops; each item gets `{event_id}:tx{i}`/`:inv{i}`/`:debt{i}`; accumulate result lists; one transaction/commit for the whole message.
- **Files:** `app/agents/agent_2_ledger.py`, `app/agents/event_schemas.py` (`LedgerUpdateData` → lists)
- **Depends on:** WP-01.
- **DoD:** compound YES records all items; duplicate replay records nothing extra; forced failure on item N rolls back all.

### WP-04 — Numbered confirmation breakdown

- **Goal:** `format_confirmation` loops N transactions / N inventory / N debts with index labels (`Transaction 1`, `Transaction 2`).
- **Files:** `app/services/formatter.py`
- **Depends on:** WP-01.
- **DoD:** breakdown for the compound message shows two transaction tables + two inventory lines, monospace.

### WP-05 — CFO reply over lists

- **Goal:** `split_routing` reply builder + warnings (low-stock/high-debt/large-expense) loop over result lists.
- **Files:** `app/agents/agent_3_cfo.py`
- **Depends on:** WP-03.
- **DoD:** post-confirm reply lists both recorded transactions and the net stock; per-item warnings fire correctly.

### WP-06 — End-to-end verification
