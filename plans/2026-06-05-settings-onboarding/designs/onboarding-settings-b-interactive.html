● Variant B — Native interactive · buttons + lists

The same flow rendered with WhatsApp's native reply buttons and list messages — the user taps instead of typing. Faster and less error-prone, but it needs new send-side code and the choices are capped (≤3 buttons, ≤10 list rows). A Phase 2 upgrade; note list/button UIs don't render on WhatsApp Web/desktop.
