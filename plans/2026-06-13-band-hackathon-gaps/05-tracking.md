- **Repo:** tali
- **Branch:** `feat/model-router`
- **Depends on:** —

- **Consumes / Produces:** Produces `G-MODEL-ROUTER` (the `get_client(role)` signature + role→provider map + fallback policy).
- **Do-not-touch:** agent files (they adopt the router in Phase 2), `band_sdk.py`.
- **DoD (self-verifiable):** unit test — `get_client("intake")`→Featherless, `get_client("cfo")`→AI/ML, forced primary-provider error→OpenAI fallback; `ai_logs` records the real provider+model+cost (Phase 1 verification #1–2).

### WP-02 — Band integration spike + connector contract

- **Goal:** Stand up `band-sdk`, register 3 remote agents on `app.band.ai` (Pro), create a room, and prove a **reply round-trips** (not just a one-way message). Decide persistent-WS vs request-scoped REST. Freezes `G-BAND-CONTRACT` incl. the **reply-collection seam** (G-05).
- **Repo:** tali
- **Branch:** `feat/band-connector`
- **Depends on:** — · **Time-box:** spike by **EOD day 2** (it gates all of Wave 1 and blocks on open-question #1).
- **Files (create/touch):** create `app/agents/band/band_client.py`, `app/agents/band/registration.md` (handles/agent_ids); touch `requirements.txt` (`+band-sdk`), `app/config.py`, `.env.example` (Band creds).
- **Consumes / Produces:** Produces `G-BAND-CONTRACT` — connector seam: send / on_message / read_context **/ reply-collection** (gateway blocks on `correlation_id`/terminal message → `send_reply`); reuses `event_schemas.py` bodies.
- **Default fallback (G-07):** if WS doesn't clearly win, adopt **request-scoped REST posting + a blocking `read_context` poll keyed by `correlation_id`** — keeps Flask synchronous, no persistent `await agent.run()`.
- **Do-not-touch:** the three agent classes (ported in Phase 2).
- **DoD (self-verifiable):** a script posts via the gateway, a 2nd agent receives via `@mention`, **and the reply is collected back and returned to a synchronous caller** (Phase 1 verification #3); WS-vs-REST decision recorded in `04`.
